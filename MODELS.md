# 🧠 AI Model Cheat Sheet (OpenCode / Go Subscription)

A quick reference for choosing the right model while working on projects like **Bingo in the Wild**.

---

## 🏆 Core Models (Use These Most Often)

### 🔹 GLM-5
**Best for:** Planning, architecture, structured thinking  
**Strengths:**
- Breaks down systems clearly
- Great TODO lists and roadmaps
- Strong backend/system design thinking

**Use when:**
- Creating TODO.md
- Designing schemas
- Planning features

---

### 🔹 Kimi K2.5
**Best for:** Deep reasoning + long context  
**Strengths:**
- Handles large code inputs well
- Connects multiple files and systems
- Strong debugging capability

**Use when:**
- Debugging complex issues
- Reviewing multiple files
- Understanding full app flow

---

### 🔹 MiniMax M2.7
**Best for:** Balanced general-purpose use  
**Strengths:**
- Good mix of reasoning + coding
- Reliable and consistent
- Handles most tasks well

**Use when:**
- You’re unsure which model to pick
- Mixed planning + coding tasks

---

### 🔹 MiniMax M2.5
**Best for:** Fast coding + iteration  
**Strengths:**
- Quick responses
- Solid coding ability
- Efficient for small tasks

**Use when:**
- Writing components/routes
- Refactoring
- Iterating quickly

---

## ⚡ Coding / Execution Models

### 🔹 Nemotron 3 Super (Free)
**Best for:** Code generation + backend work  
**Strengths:**
- Structured, clean code output
- Strong backend logic
- Less verbose, more practical

**Use when:**
- Writing Express routes
- Implementing CRUD
- Generating TypeScript interfaces

---

### 🔹 MiniMax M2.5 (Free)
**Best for:** Lightweight coding tasks  
**Use when:**
- Small fixes
- Quick iterations
- Lower-cost usage

---

## 🧪 Experimental / Secondary Models

### 🔹 Mimo V2 Omni (Free)
**Best for:** General use / experimentation  
**Strengths:**
- Flexible
- Decent all-around

**Weaknesses:**
- Less consistent

**Use when:**
- Brainstorming
- Second opinions

---

### 🔹 Mimo V2 Pro (Free)
**Best for:** Slightly more stable alternative to Omni  
**Use when:**
- Backup planning
- Alternative reasoning

---

### 🔹 Bigpickle
**Best for:** Low-stakes / experimental use  
**Use when:**
- Simple tasks
- Testing ideas

---

## 🎯 Recommended Workflow

For a project like **Bingo in the Wild**:

### 🧠 Planning
→ GLM-5

### 💻 Coding
→ Nemotron 3 Super or MiniMax M2.5

### 🐛 Debugging
→ Kimi K2.5

### 🔄 General Use
→ MiniMax M2.7

---

## 🧩 Simple Mental Model

- **GLM-5** → Tech Lead (plans everything)
- **Kimi K2.5** → Senior Engineer (solves hard problems)
- **Nemotron** → Backend Engineer (writes solid code)
- **MiniMax M2.5** → Junior Engineer (fast execution)
- **MiniMax M2.7** → Reliable generalist
- **Mimo models** → Backup thinkers
- **Bigpickle** → Intern 😄

---

## ⚡ Quick Decision Flow

1. Start with **GLM-5** → “What should I build?”
2. Switch to **M2.5 / Nemotron** → “Build this”
3. Switch to **Kimi** → “Why is this broken?”
